# Структура файлов проекта

Это эталонная структура файлов для персонального сайта-портфолио для геймдизайнера. Она основана на принципах модульности, простоты поддержки и разделения ответственности (контент, стили, ресурсы).

### 1. Визуальная схема

```
portfolio-site/
├── index.html
├── projects.html
├── research.html
├── cv.html
├── community.html
├── 404.html
│
├── assets/
│   ├── documents/
│   │   └── cv-name-surname.pdf
│   └── images/
│       ├── project-a-cover.jpg
│       └── project-b-diagram.png
│
├── css/
│   ├── base/
│   │   ├── _reset.css
│   │   ├── _variables.css
│   │   └── _typography.css
│   │
│   ├── components/
│   │   ├── _button.css
│   │   ├── _project-card.css
│   │   ├── _project-entry.css
│   │   └── _social-links.css
│   │

│   │   ├── _header.css
│   │   ├── _footer.css

│   │

│
├── .gitignore
└── README.md
```

### 2. Описание директорий и файлов

#### Корневая директория

Содержит основные HTML-страницы и файлы конфигурации проекта.

*   `index.html`: **Главная страница.** Точка входа для всех целевых аудиторий.
*   `projects.html`: **Страница проектов.** Единая страница, содержащая детальное описание всех проектов в формате последовательных секций.
*   `research.html`: **Страница исследований.** Список научных работ и публикаций.
*   `cv.html`: **Страница CV.** Подробное резюме, список навыков, опыт работы.
*   `community.html`: **Страница сообщества.** Целевая страница для аудитории, заинтересованной в поддержке и участии в жизни сообщества (Discord, Boosty, Patreon).
*   `404.html`: **Страница ошибки.** Отображается при переходе по несуществующей ссылке.
*   `.gitignore`: Файл для системы контроля версий Git, исключающий ненужные файлы из репозитория.
*   `README.md`: Данный файл. Описание структуры и назначения проекта.

#### `assets/`

Единое хранилище для всех статических медиафайлов.

*   `documents/`: Папка для PDF-файлов (резюме для скачивания, научные статьи).
*   `images/`: Папка для всех графических файлов (изображения для проектов, диаграммы, схемы, фоновые изображения).

#### `css/`

Директория со стилями, организованная по модульному принципу.

*   `style.css`: **Главный сборочный файл.** Не содержит стилей. Его единственная задача — импортировать все остальные CSS-файлы в правильном порядке.

*   **`base/`**: Глобальные стили, применяемые ко всему сайту.
    *   `_reset.css`: Сброс стандартных стилей браузера.
    *   `_variables.css`: Определение CSS-переменных (цвета, шрифты, отступы).
    *   `_typography.css`: Базовые стили для текста (`body`, `h1`-`h6`, `p`, `a`).

*   **`layout/`**: Стили для основных структурных блоков макета.
    *   `_header.css`: Стили для шапки сайта.
    *   `_footer.css`: Стили для подвала сайта.
    *   `_grid.css`: Стили для контейнеров и сеток, используемых на разных страницах.

*   **`components/`**: Стили для переиспользуемых UI-элементов.
    *   `_button.css`: Стили для кнопок.
    *   `_project-card.css`: Стили для карточек проектов (на главной).
    *   `_project-entry.css`: Стили для секции с описанием проекта (на `projects.html`).
    *   `_social-links.css`: Стили для иконок/ссылок на соцсети.

*   **`pages/`**: Специфичные стили для отдельных страниц, которые не являются компонентами.
    *   `_community-page.css`: Уникальные стили для страницы `community.html`.
    *   `_cv-page.css`: Уникальные стили для страницы `cv.html`.

### 3. Порядок импорта в `style.css`

Порядок импорта критически важен. Он следует принципу "от общего к частному", что обеспечивает предсказуемое поведение стилей и минимизирует конфликты.

```css
/* 1. БАЗОВЫЕ СТИЛИ (переменные, сброс, типографика) */
@import 'base/_variables.css';
@import 'base/_reset.css';
@import 'base/_typography.css';

/* 2. СТИЛИ МАКЕТА (сетка, хедер, футер) */
@import 'layout/_grid.css';
@import 'layout/_header.css';
@import 'layout/_footer.css';

/* 3. СТИЛИ КОМПОНЕНТОВ (переиспользуемые элементы) */
@import 'components/_button.css';
@import 'components/_project-card.css';
@import 'components/_project-entry.css';
@import 'components/_social-links.css';

/* 4. СПЕЦИФИЧНЫЕ СТИЛИ СТРАНИЦ (уникальные правила) */
@import 'pages/_community-page.css';
@import 'pages/_cv-page.css';
```

### 4. Типографика и размеры

#### Шрифты

**Основной шрифт:** `Inter` (Google Fonts)

Inter — современный, читаемый шрифт без засечек, оптимизирован для экранов. Подходит для портфолио геймдизайнера: профессиональный вид с легким современным акцентом, отлично работает на всех устройствах.

**Резервные шрифты (fallback):**
```css
font-family: 'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif;
```

**Подключение:**
```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&display=swap" rel="stylesheet">
```

#### Типографическая шкала (размеры шрифтов)

Используется модульная шкала на основе `rem` (базовая единица: `1rem = 16px`).

| Элемент | Размер (rem) | Размер (px) | Назначение |
|---------|--------------|-------------|------------|
| `body` | `1rem` | `16px` | Основной текст |
| `.text-small` | `0.875rem` | `14px` | Вспомогательный текст, подписи |
| `h6` | `1rem` | `16px` | Заголовок 6-го уровня |
| `h5` | `1.125rem` | `18px` | Заголовок 5-го уровня |
| `h4` | `1.25rem` | `20px` | Заголовок 4-го уровня |
| `h3` | `1.5rem` | `24px` | Заголовок 3-го уровня |
| `h2` | `2rem` | `32px` | Заголовок 2-го уровня |
| `h1` | `2.5rem` | `40px` | Заголовок 1-го уровня (на мобильных) |
| `.display-large` | `3rem` | `48px` | Крупный заголовок (hero-секции на десктопе) |

**Настройки строк:**
- Базовая высота строки (`line-height`): `1.6` для основного текста
- Высота строки для заголовков: `1.2` - `1.4`
- Межбуквенный интервал (`letter-spacing`): `-0.01em` для крупных заголовков

**Насыщенность шрифта (font-weight):**
- `400` (normal) — основной текст
- `500` (medium) — акценты, кнопки
- `600` (semibold) — подзаголовки
- `700` (bold) — заголовки

### 5. Цветовая схема

Цветовая палитра сайта построена на контрастной основе с акцентными красными оттенками, что создает современный и динамичный визуальный стиль, подходящий для портфолио геймдизайнера.

#### Основные цвета

| Цвет | HEX | RGB | Назначение |
|------|-----|-----|------------|
| **Основной текст** | `#F5F5F5` | `rgb(245, 245, 245)` | Цвет текста по умолчанию (светлый) |
| **Основной фон** | `#121212` | `rgb(18, 18, 18)` | Цвет фона по умолчанию (темный) |
| **Акцентный (текст/элементы)** | `#D90429` | `rgb(217, 4, 41)` | Глубокий яркий красный для акцентов, ссылок, кнопок, важных элементов |
| **Акцентный (фоны)** | `#641912` | `rgb(100, 25, 18)` | Винный темный красный для фоновых акцентов, карточек, выделенных блоков |

#### Семантические цвета

Для удобства использования и поддержки темной/светлой темы, цвета организованы через семантические переменные:

- **Цвет текста** — основной цвет для всех текстовых элементов
- **Цвет фона** — основной цвет фона страницы и контейнеров
- **Акцентный цвет** — для интерактивных элементов, ссылок, кнопок, выделений
- **Акцентный фон** — для фоновых акцентов, карточек, блоков

**Примечание:** Цвета могут инвертироваться при переключении темы (темная/светлая). Структура переменных позволяет легко менять тему через изменение значений в `:root` или добавление класса темы.

#### Использование цветов

- **Акцентный цвет (`#D90429`)** используется для:
  - Ссылок и навигации
  - Кнопок призыва к действию (CTA)
  - Важных заголовков и выделений
  - Иконок социальных сетей (при наведении)
  - Активных состояний элементов

- **Акцентный фон (`#641912`)** используется для:
  - Фонов карточек проектов
  - Выделенных секций
  - Ховер-состояний элементов
  - Фоновых декоративных элементов

### 6. Адаптивность: стопперы размеров (Breakpoints)

Для обеспечения корректного отображения на всех устройствах используются медиа-запросы с следующими точками остановки:

| Breakpoint | Размер | Назначение | Описание |
|------------|--------|------------|----------|
| `mobile` | `до 480px` | Мобильные телефоны | Вертикальная ориентация, один столбец |
| `tablet` | `481px - 768px` | Планшеты | Горизонтальная ориентация, возможно 2 столбца |
| `desktop` | `769px - 1024px` | Ноутбуки и небольшие мониторы | Стандартный десктопный макет |
| `large` | `1025px и выше` | Большие мониторы | Расширенный макет с увеличенными отступами |

**Использование в CSS:**
```css
/* Mobile First подход */
/* Стили для mobile (по умолчанию) */

/* Tablet */
@media (min-width: 481px) {
  /* Стили для планшетов */
}

/* Desktop */
@media (min-width: 769px) {
  /* Стили для десктопов */
}

/* Large Desktop */
@media (min-width: 1025px) {
  /* Стили для больших экранов */
}
```

**Рекомендации по адаптации:**
- **Мобильные (до 480px):**
  - Один столбец контента
  - Отступы: `1rem` (16px)
  - Размер заголовка H1: `2rem` (32px)
  - Компактная навигация (бургер-меню)

- **Планшеты (481px - 768px):**
  - Максимальная ширина контента: `720px`
  - Отступы: `1.5rem` (24px)
  - Возможны 2 столбца для карточек проектов
  - Размер заголовка H1: `2.5rem` (40px)

- **Десктопы (769px - 1024px):**
  - Максимальная ширина контента: `960px`
  - Отступы: `2rem` (32px)
  - Размер заголовка H1: `2.5rem` (40px)
  - Полная навигация

- **Большие экраны (1025px+):**
  - Максимальная ширина контента: `1200px`
  - Отступы: `3rem` (48px)
  - Размер заголовка H1: `3rem` (48px)
  - Увеличенные межэлементные отступы

### 7. CSS-переменные для `_variables.css`

Все перечисленные выше значения должны быть определены как CSS-переменные в файле `css/base/_variables.css` для удобства поддержки и консистентности:

```css
:root {
  /* === ЦВЕТА === */
  /* Основные цвета */
  --color-text: #F5F5F5;           /* Основной цвет текста */
  --color-bg: #121212;              /* Основной цвет фона */
  
  /* Акцентные цвета */
  --color-accent: #D90429;          /* Акцентный цвет для текста и элементов */
  --color-accent-bg: #641912;       /* Акцентный цвет для фонов */
  
  /* Дополнительные цвета (опционально, для градиентов, теней и т.д.) */
  --color-text-muted: rgba(245, 245, 245, 0.7);  /* Приглушенный текст */
  --color-border: rgba(245, 245, 245, 0.1);      /* Цвет границ */
  
  /* === ШРИФТЫ === */
  --font-family: 'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif;
  
  /* Размеры шрифтов */
  --font-size-base: 1rem;        /* 16px */
  --font-size-small: 0.875rem;   /* 14px */
  --font-size-h6: 1rem;          /* 16px */
  --font-size-h5: 1.125rem;      /* 18px */
  --font-size-h4: 1.25rem;       /* 20px */
  --font-size-h3: 1.5rem;        /* 24px */
  --font-size-h2: 2rem;          /* 32px */
  --font-size-h1: 2.5rem;        /* 40px */
  --font-size-display: 3rem;     /* 48px */
  
  /* Насыщенность шрифта */
  --font-weight-normal: 400;
  --font-weight-medium: 500;
  --font-weight-semibold: 600;
  --font-weight-bold: 700;
  
  /* Высота строки */
  --line-height-base: 1.6;
  --line-height-heading: 1.3;
  
  /* Межбуквенный интервал */
  --letter-spacing-tight: -0.01em;
  --letter-spacing-normal: 0;
  
  /* === BREAKPOINTS (для справки) === */
  /* Примечание: CSS-переменные не могут быть использованы напрямую в медиа-запросах */
  --breakpoint-mobile: 480px;
  --breakpoint-tablet: 481px;
  --breakpoint-desktop: 769px;
  --breakpoint-large: 1025px;
  
  /* === МАКСИМАЛЬНАЯ ШИРИНА КОНТЕНТА === */
  --max-width-mobile: 100%;
  --max-width-tablet: 720px;
  --max-width-desktop: 960px;
  --max-width-large: 1200px;
  
  /* === ОТСТУПЫ === */
  --spacing-xs: 0.5rem;    /* 8px */
  --spacing-sm: 1rem;      /* 16px */
  --spacing-md: 1.5rem;    /* 24px */
  --spacing-lg: 2rem;      /* 32px */
  --spacing-xl: 3rem;      /* 48px */
  --spacing-xxl: 4rem;     /* 64px */
}

/* === ПОДДЕРЖКА СВЕТЛОЙ ТЕМЫ (опционально) === */
/* Для переключения темы добавьте класс .light-theme к элементу <html> или <body> */
.light-theme {
  --color-text: #121212;           /* Инвертированный: темный текст */
  --color-bg: #F5F5F5;             /* Инвертированный: светлый фон */
  --color-text-muted: rgba(18, 18, 18, 0.7);
  --color-border: rgba(18, 18, 18, 0.1);
  /* Акцентные цвета остаются без изменений */
}
```

**Примечания:**
- CSS-переменные для breakpoints не могут быть использованы напрямую в медиа-запросах в стандартном CSS. Они указаны для справки и документирования. В медиа-запросах используйте значения напрямую (например, `@media (min-width: 481px)`).
- Для переключения между темной и светлой темой используйте класс `.light-theme` на корневом элементе. Акцентные цвета (`--color-accent` и `--color-accent-bg`) остаются неизменными в обеих темах.
